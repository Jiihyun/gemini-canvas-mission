# gemini-canvas-mission

Gemini Canvas 미션 결과물을 관리하는 저장소입니다.

## 목적

- Gemini Canvas로 제작한 웹앱 결과물을 기록하고 공유합니다.
- 앱별 요구사항, 배포 링크, 회고를 일관된 형식으로 남깁니다.

## 💻 1. 유틸리티 앱

### 앱 이름

Snap Clean

### 배포 링크

https://gemini.google.com/share/47ef71310f52

### 이 앱을 만든 이유

- 어떤 문제/불편함을 해결하려고 했나요?
    - 아이폰 기본 사진 앱에서 연사나 비슷한 사진들을 정리할 때, 
    사진을 일일이 비교하고 여러 번의 터치를 거쳐 삭제해야 하는 번거로움과 '결정 피로도'가 큽니다. 
    이로 인해 **사진 정리를 계속 미루게 되고 결국 휴대폰 용량 부족으로 이어지는 악순환을 해결**하고자 했습니다.
- 본인이 이 앱을 언제, 어떻게 사용할 건가요?
    - **여행이나 특별한 이벤트 직후:** 대중교통으로 돌아오는 길이나 자투리 시간에 앱을 켜서, 그날 연사로 찍은 수십 장의 비슷한 사진들을 빠르게 쳐낼 때 사용할 예정입니다.
    - **휴대폰 저장 공간이 부족할 때:** 갤러리를 무작정 뒤지는 대신, SnapClean이 모아준 '정리 가능한 묶음'만 빠르게 확인하고 불필요한 사진을 비워내 용량을 즉각적으로 확보하려 합니다.
    - **평소 갤러리 정리를 가볍게 하고 싶을 때:** 키보드 방향키를 통해 남길 사진만 '좋아요'를 누르며 스트레스 없이 갤러리를 가볍게 만들 때 사용할 예정입니다.

### 주요 기능

- **남길 것만 고르는 초고속 분류**
    - 삭제할 사진을 고르는 대신, 유지할 사진(베스트 샷)에만 '좋아요(키보드 `↓`버튼)'를 누르고 나머지는 빠르게 패스(키보드`→` 버튼)하는 방식을 적용했습니다.
- **심리적 안전장치(안심 삭제 후보함)**
    - 정리를 마친 사진들이 즉시 영구 삭제되지 않고 '삭제 후보함'으로 이동하여 30일간 보관됩니다. 실수로 지웠더라도 언제든 원클릭으로 복구할 수 있어, 사용자가 삭제에 대한 불안감 없이 망설임 없이 정리를 진행할 수 있게 돕습니다.

### AI 기능 (해당되는 경우)

- 어떤 AI 기능을 추가했나요?
    - 촬영 시간과 구도 기반의 **'유사 사진 자동 묶음(그룹화)'** 기능을 추가했습니다.
- AI 기능이 앱에서 어떤 역할을 하나요?
    - 사용자가 직접 비슷한 사진을 찾아 헤맬 필요 없이 정리가 필요한 사진들을 하나의 묶음으로 만들어줘서, 필요없는 사진을 빠르게 정리하는 것을 도와주는 역할을 합니다.

## 🎮 2. 게임

### 앱 이름

Voca Catch

### 배포 링크

https://gemini.google.com/share/dd185a917cb9

### 이 앱을 만든 이유

- 어떤 문제/불편함을 해결하려고 했나요?
    - **캡처 화면의 '무덤화' 방지:** 유용한 영어 표현을 발견할 때마다 화면을 캡처하지만, 정작 갤러리에 쌓여만 가고 다시 열어보지 않는 문제를 해결하고자 했습니다.
    - **입력의 번거로움 제거:** 사진 속 문장을 일일이 타이핑하여 단어장에 옮기는 과정은 매우 귀찮은 일입니다. AI 기능을 통해 영어 표헌을 추출하여 저장할 수 있는 경험을 제공하여 학습의 진입장벽을 낮췄습니다.
    - **지루한 암기 방식의 탈피:** 단순히 텍스트를 읽는 방식에서 벗어나, 긴박감 넘치는 캐치 게임 요소를 결합하여 암기 자체를 '보상형 재미'로 전환했습니다.
- 본인이 이 앱을 언제, 어떻게 사용할 건가요?
    - 출퇴근 지하철이나 버스 안에서 유튜브를 보는 대신, 내가 직접 캡처했던 실전 표현들을 게임으로 복습하며 시간을 생산적으로 활용할 것입니다.

### 주요 기능

- **원클릭 AI OCR 스캔**
    - 이미지 내의 복잡한 영어 문장과 한국어 해석을 AI가 한 번에 찾아내어 단어와 뜻으로 구조화합니다. 사용자가 별도로 편집할 필요 없이 즉시 학습 데이터로 변환됩니다.
- **캐치 게임 스타일의 암기 방식**
    - 하늘에서 떨어지는 여러 뜻 중에서 상단 영어 표현과 일치하는 정답을 바구니에 담아 점수를 얻을 수 있는 게이미피케이션 학습 환경을 제공합니다. 시각적 긴장감이 뇌의 기억 회로를 자극하여 암기 효율을 높입니다.

### AI 기능 (해당되는 경우)

- 어떤 AI 기능을 추가했나요?
    - **Google Gemini 2.5 Flash 기반 멀티모달 분석:** 단순한 글자 인식을 넘어, 이미지의 맥락을 이해하고 '영어'와 '한국어 의미'를 정확히 매칭하는 고급 AI 모델을 탑재했습니다.
- AI 기능이 앱에서 어떤 역할을 하나요?
    - 비정형 데이터인 이미지에서 텍스트를 추출한 뒤, 앱이 바로 사용할 수 있는 **JSON 형식**으로 변환해줍니다. 이를 통해 사용자는 수동 입력 없이 즉시 게임 콘텐츠를 생성할 수 있습니다.

## 📚 3. 학습 앱

### 앱 이름

BOJ Algo Visualizer

### 배포 링크

https://gemini.google.com/share/59699e5a5279

### 이 앱을 만든 이유

- 어떤 문제/불편함을 해결하려고 했나요?
    - 백준(BOJ) 등 코딩테스트 문제를 풀고 난 후, 정답 코드를 보더라도 그 내부 로직이 어떻게 동작하는지 머릿속으로만 따라가기에는 큰 어려움이 있습니다. 이러한 이해의 장벽을 낮추고, 복잡한 알고리즘의 동작 과정을 눈으로 직접 보며 직관적이고 쉽게 이해하기 위해 'BOJ 알고리즘 시각화 가이드'를 만들었습니다.
- 본인이 이 앱을 언제, 어떻게 사용할 건가요?
    - 코딩테스트 문제 풀이 및 리뷰 시간에, 다른 사람의 우수한 정답 코드나 내가 짰지만 동작 흐름이 완벽히 와닿지 않는 코드를 마주했을 때 사용할 예정입니다.
    이해 안 가는 Java 코드를 앱에 복사해 넣고, 재생 버튼을 눌러 단계별 변수 값의 변화와 데이터 구조(배열/격자)의 움직임을 시각적으로 천천히 따라가며 알고리즘의 원리를 체화하는 용도로 활용할 것입니다.

### 주요 기능

- **코드 시각화 엔진**: 코딩테스트 문제의 정답 코드(Java)를 입력하면, 배열이나 2차원 격자의 변화를 시각적인 블록 요소로 렌더링하여 보여줍니다.
- **단계별 애니메이션 제어**: 재생, 일시정지, 배속 조절(0.5x, 1.0x, 2.0x) 기능을 통해 원하는 시점의 동작을 세밀하게 관찰할 수 있습니다.
- **실시간 디버깅 창**: 실제 IDE 환경처럼 현재 실행 중인 단계의 설명, 그리고 활성화된 지역 변수들의 상태를 표형태로 실시간 추적하여 보여줍니다.

### AI 기능 (해당되는 경우)

- 어떤 AI 기능을 추가했나요?
    - Gemini 2.5 Flash를 이용하여 정적 코드를 동적 애니메이션 데이터로 변환하는 '코드 분석 및 시각화 데이터 생성 엔진'을 구축했습니다.
- AI 기능이 앱에서 어떤 역할을 하나요?
    - 코드가 어떤 알고리즘인지 파악하고, 전체 흐름을 사용자가 이해하기 쉽도록 스텝을 쪼개어 알고리즘 튜터의 역할을 담당합니다.

### 2. 프롬프트 개선 과정

- 처음 결과물의 문제점은 무엇이었나요?
    - **정보의 과잉:** 사진을 찍은 날짜, 묶음의 제목, AI 분석 뱃지 등 부가적인 정보가 많아 '초고속 분류'라는 핵심 목적에 집중하기 산만했습니다.
    - **테스트 환경과의 불일치:** 모바일 앱을 상정하고 '스와이프 제스처'를 구현했지만, 실제 결과물을 확인하는 환경이 PC 웹 브라우저였기 때문에 직접 만져보고 테스트하기가 어려웠습니다.
    - **사용자 플로우 단절:** 액션(좋아요/삭제)을 취했을 때 자동으로 다음 사진으로 넘어가지 않아, 사용자가 일일이 다음 버튼을 눌러야 하는 불편함이 있었습니다.
- 어떤 식으로 프롬프트를 수정했나요?
    - **이미지 레퍼런스 활용:** 말로 설명하기 까다로운 레이아웃은 "사진과 같은 디자인을 적용해줘"라며 사진 이미지를 업로드하여 시각적 의도를 정확히 전달했습니다.
    - **정확한 디버깅 및 대안 제시:** 단순히 “잘 안돼”라고 말하며 에러를 해결해달라는 말 대신에 “~할 때 ~가 안 되니 ~로 바꿔줘”와 같이 대안 또는 원하는 바를 전달했습니다.
    - **환경에 맞춘 입력 방식 변경:** 웹 환경에서 제스처 확인이 어렵자 "키보드 화살표 위아래 양옆으로 적용해줘"라며 입력 방식을 우회하고, "눌렀을 때 상태가 변하면서 다음 사진으로 넘어가야해"와 같이 세부 동작 플로우를 덧붙였습니다.
    - **기술 스택 제한:** 디자인과 로직이 어느 정도 완성된 후 "기능은 그대로 유지하되, HTML, CSS, JavaScript만 사용해줘. React나 다른 프레임워크는 사용하지 마"라고 제한을 두어 원하는 최종 코드 포맷을 얻어냈습니다.
- 몇 번의 반복을 거쳤나요?
    - 초기 기획안부터 불필요한 텍스트/뱃지 제거, 키보드 인터랙션 추가, 순수 JS로의 변환, 그리고 PC 사진 불러오기 로직 추가까지 약 15번 안팎의 프롬프트 반복을 거치며 프로토타입을 정교화했습니다.

### 3. 효과적이었던 프롬프팅 전략

- 어떤 방식으로 요청했을 때 원하는 결과가 나왔나요?
    - **명확하고 구체적인 레퍼런스 제시:** "아이폰 사진 앱처럼 지금 보는 사진 밑에 다른 사진들도 보여주고..."처럼 누구나 아는 구체적인 레퍼런스를 지목했을 때, 긴 설명 없이도 원하는 형태의 썸네일 네비게이션이 한 번에 구현되었습니다.
    - **참고 이미지 직접 제공:** 연속 사진이 펼쳐진 느낌을 텍스트로 길게 묘사하는 대신, 이미지를 한 장 던져주었을 때 AI가 CSS 애니메이션과 레이아웃수치를 훨씬 정확하게 잡아냈습니다.
- 반대로 효과가 없었던 방식은 어떤게 있었나요?
    - **추상적이고 모호한 단어 사용: “좀 더 재밌는 게임이었으면 좋겠어”나 “관련 없는 배경 지워줘”** 같이 모호한 단어를 사용해 추상적으로 요청하니, 사람마다 ‘재미’, ‘관련’의 기준이 다르기 때문에 제가 원하는 느낌의 구현이 나오지 않았습니다.

### 4. 배운 점

- Gemini Canvas를 효과적으로 사용하기 위한 나만의 팁이 있다면?
    - 처음 프롬프트에서는 *무엇을 만들 것인지*에 집중해 핵심 가치와 기술 스택을 명확히 정의하는 것이 중요하다고 생각합니다. 이후에는 프로토타입을 기반으로 사용자 경험을 구체화하고, 논리적 허점을 보완하며 전체적인 밸런스를 조정하는 방식이 효과적인 것 같습니다. 즉 단계별로 문제를 해결해 나가는 방식이 원하는 결과를 얻기 쉽다고 생각합니다.
    - 프롬프트를 작성할 때는 최대한 구체적으로 작성하고, 사람마다 기준이 달라질 수 있는 부분은 내가 원하는 바를 명확히 표현하는 것이 중요하다고 생각합니다. 만약 어떤 부분이 모호하게 작성되었는지 스스로 인지하지 못했다면, AI에게 임의로 결정하지 말고 어느 부분이 모호하게 느껴지는지 되물어달라고 요청하는 것이 좋습니다. 이렇게 하면 불필요한 수정 과정을 줄이고, 보다 원하는 방향에 가깝게 구현할 수 있다고 생각합니다.
